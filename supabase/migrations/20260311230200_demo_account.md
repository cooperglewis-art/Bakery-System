# Demo Account Setup

Supabase auth accounts cannot be created via SQL migrations in the public schema.
The `auth.users` table is managed internally by Supabase Auth (GoTrue) and inserting
directly is unsupported and fragile.

## Recommended approaches

1. **Supabase Dashboard** -- Go to Authentication > Users > "Add user" and create:
   - Email: `demo@bakery-orders.example`
   - Password: (choose a password for the demo account)

2. **Supabase CLI seed** -- Use `supabase/seed.sql` with the management API, or
   invoke the signup endpoint in a seed script:
   ```bash
   curl -X POST "$SUPABASE_URL/auth/v1/signup" \
     -H "apikey: $SUPABASE_ANON_KEY" \
     -H "Content-Type: application/json" \
     -d '{"email": "demo@bakery-orders.example", "password": "demo-password-123"}'
   ```

3. **Application signup flow** -- Simply sign up through the app's normal registration page.
